## Overview

A F provides a centralized space to create, organize, and manage all your project documentation. You can build structured docs sites with MDX support, version history, and seamless collaboration. Start organizing your technical guides, API references, and user manuals today.

## Getting Started

Follow these steps to create your first documentation page.

<Steps>
  <Step title="Create a New Space" icon="plus">
    Log in to your A F dashboard at `https://dashboard.example.com` and click **New Space**.

    Select your project and configure basic settings like name and description.
  </Step>
  <Step title="Add Your First Page" icon="file-text">
    Navigate to the **Pages** section and create `introduction.mdx`.

    Use the built-in editor to add frontmatter and content.
  </Step>
  <Step title="Preview and Publish" icon="eye">
    Preview your page with live MDX rendering, then publish to make it live.
  </Step>
</Steps>

Here is a basic code example for your first MDX page:

<CodeGroup tabs="MDX,Bash">
  ```mdx
  ---
  title: My First Page
  description: Welcome to my docs.
  ---

  ## Hello World

  This is your first documentation page.
  ```
  ```
  bash
  # Upload via CLI
  af upload introduction.mdx --space=my-project
  ```
</CodeGroup>
